# Shortcut Keys Documentation

This document outlines the shortcut keys used in the project workflow.

## Shortcut Keys

| Key | Description |
|-----|-------------|
| R   | Revert to immediate git commit |
| N   | Never save |
| C   | Commit After Change |
| S   | Save all changes |

## Usage

These shortcuts are designed to streamline the development workflow by providing quick access to common version control and file operations.

### R - Revert to immediate git commit
- Discards all current changes
- Restores files to the state of the last git commit
- Useful when you want to start over with a clean slate

### N - Never save
- Prevents saving any changes made during the session
- Changes are kept in memory but not written to disk
- Useful for experimental work that you don't want to persist

### C - Commit After Change
- Automatically commits changes after each modification
- Helps maintain a detailed version history
- Useful for tracking incremental progress

### S - Save all changes
- Saves all current changes to disk
- Does not create a git commit
- Useful for preserving work between sessions
